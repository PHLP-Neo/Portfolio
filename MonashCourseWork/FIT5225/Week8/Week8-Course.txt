pg 58

different routing methods

Simple Routing:
This is the most basic routing method. A domain name is mapped directly to a single resource (like an IP address or a web server) without any intelligent traffic distribution. It's used when you have only one resource to route traffic to.

Weighted Routing:
In this method, you can assign weights to different resources. Traffic is distributed across multiple resources based on the specified weights. For example, if you want to send 70% of the traffic to one server and 30% to another, you can configure it using weighted routing.

Latency-based Routing:
This method sends traffic to the resource that has the lowest network latency (i.e., the fastest response) for the user. It's useful for improving application performance globally, by directing users to the fastest available endpoint.

Failover Routing:
Failover routing is used to automatically route traffic to a healthy resource if the primary resource becomes unhealthy. It helps in building highly available systems by switching to a backup if the main system fails.

Geolocation Routing:
In this method, users are routed based on their geographic location. For example, users from Europe can be directed to European servers, and users from Asia can be sent to Asian servers.

Geo-proximity Routing:
Geo-proximity routing routes traffic based on the geographic location of users and the proximity to AWS resources. You can also bias routing to increase or decrease the size of the region from which traffic is routed to a resource.

Multivalue Answer Routing:
When a user queries the DNS, multiple healthy IP addresses are returned. This is somewhat like simple routing but with basic load balancing and health checks built-in - if one IP becomes unhealthy, it is automatically removed from the responses.
