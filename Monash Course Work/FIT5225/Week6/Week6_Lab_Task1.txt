$ sudo pip3 install -U ansible openstacksdk openstackclient && sudo ansible-galaxy collection install openstack.cloud
First half:
 - U: Upgrade(so if you already installed, will check for upgrade instead)
Second half:
 - ansible-galaxy: user terminal for ansible, you can use it as an appstore to download differet ansible plugin and add-on
 - collection: a file format for ansible module, plugins and roles
 - openstack.cloud: the add-on we want to install

Ansible does not natively support communication with nectar, so we need to install corresponding add-on first.

basically: 
first half: install ansible and openstack
second half: install plugin for ansible, so it can communicate with openstack

10. create hosts.ini file

[web_servers]
SERVER_1_IP_HERE
SERVER_2_IP_HERE 

find server ip address from nectar dashboard is fine i think

$ ansible -i hosts.ini -v web_servers -m ping --private-key=$PRI_KEY_PATH -u ubuntu
 -i hosts.ini: inventory, nectar need this to know who to send
 -v verbose: if fails, print detailed logs (for debug)
 -m ping: use ping module from Ansible, to ensure it can ssh to the remote mathine, check for python and remote control
 --pricate-key=$PRI_KEY_PATH: need private key for ssh
 -u ubuntu, basically the first half of the ubuntu:<ip-address>

11.
run this line:
$ ansible -i hosts.ini -v web_servers -m apt -a "name=apache2 state=latest" --private-key=$PRI_KEY_PATH -u ubuntu --become
 -m apt: use module apt, the same "apt" in "sudo apt install", to manage package installation
 -a "...": arguments, what you want to do with apt module.
 name=apache2: tell apt to install Apache2
 state=latest: tell apt if the apache2 does not exist, install it, if exist, check for update
 --become: tell Ansible to add "sudo" when running apt command
 
Idempotency:
 - since we have state=latest, the program will not crash no matter how many times it runs, even if it is already installed on the server.



use this command to start apache http service:
ansible -i hosts.ini -v web_servers -a "systemctl start apache2" --private-key=$PRI_KEY_PATH -u ubuntu --become
