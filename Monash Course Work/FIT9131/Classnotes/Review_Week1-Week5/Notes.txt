2.3.3

The scope of a variable defines the section of source code from which the variable can be accessed.
There are two scope: class scope and block scope

A block can be placed anywhere in the code; however, they should only be used when the syntax mandates the use of a block. Some of the most common places to define a block include:
body of a class
body of a method
selection constructs (we will see these later)
loops (we will see these later as well).

Fields
-Definition: Defined outside constructors and methods.
-Scope: Within any methods or constructors of the class in which they are defined.

Parameters
-Definition: Defined in the header of the method or constructor.
-Scope: Within the body of the method in which they are defined.

Local variables
-Definition: Defined within the body of a method or constructor.
-Scope: Within the block in which they are defined, starting from the line where they are defined.

2.3.8

All primitive data types are passed between methods using a concept known as pass by value. When a primitive variable is passed as an argument, a copy of that variable is made and the copy is then passed to the subsequent method. The implication of this is that any modifications made to the copy of the variable do not impact the original variable. Let's see an example of this in code using the Car example from earlier.

All object data types are passed between methods using a concept known as pass by reference and as such, object variables are sometime referred to as reference variables. Unlike primitive types that have a fixed size of memory allocated to the variable, object type variables are often more complex, comprising multiple fields. It can be very cumbersome to pass these complex objects around. As such, Java uses a concept whereby a reference or address of the object is passed around rather than the object itself.

public class Car
{
    private String make;
    private String model;
    private int year;
    private String color;

    public Car()
    {
        make = "Unknown";
        model = "Unknown";
        year = 1900;
        color = "Unknown";
    }

    public Car(String newMake, String newModel, int newYear, String newColor)
    {
        make = newMake;
        model = newModel;
        year = newYear;
        color = newColor;
    }

    public void display()
    {
        System.out.println(year + " " + color + " " + make + " " + model);
    }

    public String getColor()
    {
        return color;
    }

    public String getMake()
    {
        return make;
    }

    public String getModel()
    {
        return model;
    }

    public int getYear()
    {
        return year;
    }

    public static void main(String[] args)
    {
        Car objCar = new Car("Lamborghini", "Diablo", 2000, "Yellow");
        System.out.println("Lets make a copy of the car object now in a new variable");
        Car objCar2 = objCar;
        System.out.println("Lets call the display() method using the first object");
        objCar.display();
        System.out.println("Lets call the display() method using the second object");
        objCar2.display();
        System.out.println("Modifying the second object, changing the model only");
        objCar2.setModel("Aventador");
        System.out.println("Lets call the display() method using the first object again");
        objCar.display();
        System.out.println("Lets call the display() method using the second object again");
        objCar2.display();
        System.out.println("Note how each object variable is NOT a copy but actually point to the same object in memory");
    }

    public void setColor(String newColor)
    {
        color = newColor;
    }

    public void setMake(String newMake)
    {
        make = newMake;
    }

    public void setModel(String newModel)
    {
        model = newModel;
    }

    public void setYear(int newYear)
    {
        year = newYear;
    }
}
